Λαμβάνοντας υπόψη τις ατελείωτες συζητήσεις σχετικά με τη βασική λειτουργία του Robonomics, ας διερευνήσουμε τη δομή κάτω από την επιφάνεια και ας σκιαγραφήσουμε τους βασικούς πυλώνες του δικτύου. Αυτό το άρθρο εμβαθύνει στη σύνδεση μεταξύ της πλατφόρμας Robonomics, του [parachain](https://blog.aira.life/robonomics-parachain-on-kusama-40853780c709) της στο οικοσύστημα Polkadot και των [Robonomics Web Services](https://blog.aira.life/robonomics-web-services-and-rws-token-intro-d730ab50ad42) (RWS).

Το Robonomics Network αποτελείται από πολλαπλά [αποθετήρια κώδικα](https://github.com/airalab) που μπορούν να χρησιμοποιηθούν ξεχωριστά για την υλοποίηση μιας πλήρους εφαρμογής IoT. Αν, για παράδειγμα, σκοπεύετε να συνδέσετε το smartphone σας με μια συσκευή IoT, δεν αρκεί να αναπτύξετε μια εφαρμογή και να συνδέσετε μερικούς αισθητήρες σε αυτήν. Αντίθετα, πρέπει να εκτελέσετε μια [αλυσίδα ενεργειών](https://blog.aira.life/simple-iot-scheme-990224363803) που τελικά θα οδηγήσει σε μια συνδεδεμένη συσκευή IoT και διασυνδεδεμένα στοιχεία της υποδομής, επιτρέποντας το σήμα από τη συσκευή να φτάσει σε εσάς.

## Η χρονολογία της ανάπτυξης του Robonomics Network

Πώς λοιπόν προέκυψε η ιδέα για τη δημιουργία του [Robonomics parachain](https://parachain.robonomics.network/#/explorer) και του [Robonomics Network](https://robonomics.network/land/say-hello-to-robonomics/); Αρχικά, πίσω στο 2015–2017, ξεκινήσαμε με την ανάπτυξη ενός αρκετά πολύπλοκουλύση που ονομάσαμε [Οικονομία των Ρομπότ](https://blog.aira.life/the-first-in-the-economy-of-robots-6f4bb115643b). Είμαστε βέβαιοι ότι είναι ακριβώς αυτό που θα χρειαστούν οι έξυπνες πόλεις, η βιομηχανία 4.0 και η ανάπτυξη εφαρμογών IoT.

Ο κύριος στόχος μας ήταν να δημιουργήσουμε λογισμικό για κυβερνο-φυσικά συστήματα που μπορούν να διαχειρίζονται αυτόνομες και ανεξάρτητες συσκευές στο πλαίσιο τεχνικών και οικονομικών συναλλαγών. Επομένως, αποφασίσαμε να δημιουργήσουμε μια τεχνική και οικονομικά προσανατολισμένη συναλλαγή και να την κάνουμε θεμέλιο για την ανάπτυξη πιο πολύπλοκων διαδικασιών.

Ένα από τα αγαπημένα μου σενάρια της Οικονομίας των Ρομπότ ονομάζεται [Smart Building Offsetting](https://new.dapp.ipci.io/#/offsetting). Είναι μια υπηρεσία που επιτρέπει σε ένα αυτόνομο κτίριο να συλλέγει και να συγκεντρώνει πληροφορίες για την κατανάλωση ενέργειας του και στη συνέχεια να τις επικοινωνεί σε έναν άλλο πράκτορα εντός του Robonomics Network, και ακόμη και να ζητά υπηρεσίες αντιστάθμισης του ανθρακικού αποτυπώματος. Με τη σειρά του, ο πράκτορας επαληθεύει την τοποθεσία του κτιρίου, υπολογίζει έναν κατάλληλο συντελεστή εκπομπών CO2 και στη συνέχεια συγχρονίζει αυτά τα δεδομένα με το συνεργαζόμενο έργο [DAO IPCI](http://ipci.io/), για να καθορίσει τον αριθμό των πιστώσεων άνθρακα από την αγορά πράσινης χρηματοδότησης που απαιτούνται για την αντιστάθμιση του ανθρακικού αποτυπώματος του κτιρίου. Αυτές οι λειτουργίες εκτελέστηκαν με επιτυχία στο blockchain του Ethereum.

### Η λύση στο πρόβλημα

Σήμερα, η Οικονομία των Ρομπότ που καταφέραμε να χτίσουμε το 2017 με τους συναδέλφους και τους συνεργάτες μας, αποτελεί μια περίπλοκη αλυσίδα με την πρώτη ματιά, που λειτουργεί ως το τελικό στάδιο της εξέλιξης της βιομηχανίας IoT. Το μεγαλύτερο πρόβλημα που είχαμε ήταν ότι συσσωρεύαμε όλη τη γνώση και υλοποιούσαμε τις ιδέες μας, χωρίς να έχουμε τη δυνατότητα να τις μεταδώσουμε σε κανέναν. Κάθε νέο [σενάριο της Οικονομίας των Ρομπότ](https://wiki.robonomics.network/docs/r-and-d-based-on-robonomics-network/), που δημιουργήθηκε κατά την ανάπτυξη της πλατφόρμας Robonomics Network, έκανε το τελικό προϊόν λίγο πιο δύσκολο να κατανοηθεί από ανθρώπους που δεν είναι εξοικειωμένοι με το αντικείμενο. Με άλλα λόγια, κάποιος πρέπει να είναι εξοικειωμένος με πολλαπλές δεξιότητες και να αφιερώσει χρόνια για να κατανοήσει τη σκέψη πίσω από τις διασταυρούμενες επιστήμες που διαμορφώνουν τις έννοιες της οικονομίας και των μηχανών που συνεργάζονται.

Όταν αρχίζεις να κατανοείς τις λεπτομέρειες πίσω από την ανάπτυξη μιας εφαρμογής IoT, γίνεται γρήγορα φανερό ότι οι τεχνικοί ειδικοί πρέπει να αφιερώσουν χρόνια για να φτάσουν στο στάδιο ενός πλήρως λειτουργικού προϊόντος. Γι' αυτό αποφασίσαμε να μετατρέψουμε τους πιθανούς χρήστες της πλατφόρμας μας σε συν-δημιουργούς. Εστιάσαμε στο να τους διδάξουμε να γίνουν επαγγελματίες μηχανικοί πληροφορικής στον χώρο των εφαρμογών IoT και ωςως αποτέλεσμα, δημιουργήσαμε το Robonomics parachain και τις Robonomics Web Services.

## Εισαγωγή στις Robonomics Web Services

Είναι σαφές ότι η Οικονομία των Ρομπότ ενδιαφέρει το κοινό. Παρ' όλα αυτά, το κύριο πρόβλημα είναι ότι μετά από λίγο καιρό αυτοί οι χρήστες καταλήγουν να χρησιμοποιούν τις Amazon Web Services για συσκευές IoT και αρχίζουν να τις χρησιμοποιούν αντί γι' αυτό, χωρίς να αντιλαμβάνονται τα κρίσιμα πλεονεκτήματα που προσφέρει το Robonomics Network σε σύγκριση με τους γνωστούς γίγαντες της πληροφορικής. Αυτό μας οδήγησε στο συμπέρασμα ότι δεν μπορούμε να μιλάμε μόνο για υποσχέσεις ολοκλήρωσης κάποιου έργου στο μέλλον χωρίς να παρέχουμε ένα πραγματικό προϊόν εδώ και τώρα.

Γι' αυτό αποφασίσαμε ότι πρέπει να προσφέρουμε μια σειρά από πραγματικές υπηρεσίες στους δυνητικούς μας χρήστες. Υπηρεσίες παρόμοιες με αυτές που προσφέρουν άλλες πλατφόρμες cloud, αλλά καλύτερες. Ως αποτέλεσμα, δημιουργήσαμε τις Robonomics Web Services, ένα προϊόν βασισμένο στα δικά μας πρωτόκολλα και πλατφόρμα που χρησιμοποιεί ένα ανοιχτού κώδικα ρομποτικό πλαίσιο (ROS). Στη συνέχεια, αφιερώσαμε 2 χρόνια στην ανάπτυξη των βασικών λειτουργιών, στις οποίες θα εμβαθύνω τώρα με περισσότερες λεπτομέρειες.

### Οι βασικές λειτουργίες των RWS:

**1. Συνδρομή**

Η τυπική προσέγγιση στις υπηρεσίες blockchain περιλαμβάνει πάντα κόστη συναλλαγών γνωστά ως τέλη συναλλαγής ή gas συναλλαγής. Αυτό σημαίνει ότι όταν κάνετε μια συναλλαγή, η προτεραιότητά της εξαρτάται από το κόστος ή το τέλος που είστε διατεθειμένοι να πληρώσετε γι' αυτήν. Όσο υψηλότερο είναι το τέλος που πληρώνετε, τόσο πιο γρήγορα θα διεκπεραιωθεί η συναλλαγή σας. Ένα τέτοιο μοντέλο δεν είναι κατάλληλο για συστήματα που βασίζονται σε εγγυημένο εύρος ζώνης. Πρέπει να είστε απολύτως σίγουροι ότι η συσκευή σας θα μπορεί να πραγματοποιήσει μια συναλλαγή στον προκαθορισμένο χρόνο και με προβλέψιμο κόστος. Αυτό είναι κάτι που δεν βλέπουμε συχνά σήμερα, καθώς τα κόστη συναλλαγών εξαρτώνται πλήρως από τις διακυμάνσεις των συναλλαγματικών ισοτιμιών, φτάνοντας κάποιες φορές τα $50.

Για να λύσουμε αυτό το ζήτημα, δημιουργήσαμε την υπηρεσία Συνδρομής, η οποία αποτελεί ένα εναλλακτικό μοντέλο κόστους στο Robonomics parachain για το Polkadot. Εγγυάται την εκτέλεση ενός προπληρωμένου αριθμού συναλλαγών και τους αποδίδει υψηλότερη προτεραιότητα σε σύγκριση με τις συναλλαγές που πληρώνονται μέσω του μοντέλου με τέλη. Για να μάθετε περισσότερα για τη διαφορά μεταξύ της μηνιαίας Συνδρομής και των XRT tokens μπορείτε να ανατρέξετε στο άρθρο μου ‘[Heterogeneous tokenomics](https://blog.aira.life/rws-overview-part-2-heterogeneous-tokenomics-afc209cc855)’, ενώ ένα άλλο άρθρο μου που προγραμματίζεταιπου θα κυκλοφορήσει στο τέλος Δεκεμβρίου θα περιγράφει την τελική αρχιτεκτονική και τη διασύνδεση των δύο μοντέλων.

Το μοντέλο Συνδρομής μας δεν λύνει μόνο το πρόβλημα της εγγυημένης μετάβασης, αλλά κοστίζει επίσης μόνο $10 ανά συσκευή, καθιστώντας το μια εξαιρετικά κερδοφόρα προσφορά για τον τελικό χρήστη. Επιπλέον, παρέχει σταθερό εύρος ζώνης για χιλιάδες μηνιαίες συναλλαγές, κάτι που αποτελεί πραγματικά ένα εντυπωσιακό επίτευγμα.

**2. Εκκίνηση**

Η Εκκίνηση είναι μια πολύ απλή αλλά οικονομική λειτουργία από άποψη χρήσης υπολογιστικής ισχύος και blockchain. Σας επιτρέπει να ενεργοποιείτε ή να απενεργοποιείτε τη συσκευή σας και να αλλάζετε τη δυαδική της ρύθμιση από 1 σε 0 και αντίστροφα, στέλνοντας ένα αίτημα με τη μορφή συναλλαγής. Δεδομένου ότι δεν υπάρχει μεγάλη πολυπλοκότητα σε αυτή τη λειτουργία, ας προχωρήσουμε στον επόμενο κρίσιμο πυλώνα.

**3. Datalog**

Αυτή η λειτουργία σχετίζεται με την τηλεμετρία και γι' αυτό χρειάζεται να επεκταθώ σε αυτήν. Λειτουργεί σε δύο λειτουργίες, όπου είτε αποθηκεύετε τα IPFS hashes σας είτε αποθηκεύετε δεδομένα στην αλυσίδα.

Πώς χρησιμοποιείτε αυτές τις λειτουργίες; Η λειτουργία datalog στην αλυσίδα εφαρμόζεται όταν το μέγεθος των δεδομένων σας είναι κάτω από τέσσερις μεταβλητές. Δεν έχει νόημα να αποθηκεύσετε ένα τόσο μικρό κομμάτι δεδομένων στο IPFS (εκτός αλυσίδας), καθώς σε επίπεδο bytes αυτό το log στην αλυσίδα δεν θα είναι σημαντικά μεγαλύτερο από το IPFS hash. Προτείνουμε να καταγράφετε ολόκληρη τη ροή ακατέργαστων δεδομένων ως μία λειτουργία ή να επιλέγετε το IPFS hash. Στη συνέχεια, το Robonomics ως πλατφόρμα για προγραμματιστές θα επεξεργάζεται τα καταγεγραμμένα δεδομένα, είτε είναι ακατέργαστα είτε με τη μορφή IPFS hash, και θα αποφασίζει πού θα τα γράψει. Αυτή η επεξεργασία datalog θα είναι κρυφή από τον τελικό χρήστη και δεν θα παρατηρήσει καν τη διαφορά μεταξύ των δύο διαφορετικών τύπων αποθήκευσης.

Το ‘Liability’ είναι η τελική μας λειτουργία, αλλά δεν θα κυκλοφορήσει στο κοινό μέχρι να γνωρίζουμε τους στόχους που επιδιώκουν τα μεγάλης κλίμακας έργα Οικονομίας των Ρομπότ. Ένα ακόμη σημαντικό προϊόν για εμάς, στο οποίο εργαζόμαστε, στοχεύει στο πλαίσιο [Robot as a Service](https://blog.aira.life/what-is-robonomics-network-robot-as-a-service-and-future-parachain-of-polkadot-or-something-more-36deece9f3cb), ή όπως το αποκαλούμε, το ‘συμβατικά υποχρεωμένο ρομπότ’ στο Ethereum. [Αυτή η υποδομή](https://blog.aira.life/robots-under-robonomics-control-overview-39337ac9fbbb) έχει ήδη περιγραφεί με κάθε λεπτομέρεια στο knowledge hub μας και υπονοεί μια κατάσταση όπου ο τελικός χρήστης χρησιμοποιεί τα service tokens μαςχωρίς να βασίζεστε σε XRT ή άλλα tokens για την πληρωμή της χρήσης του parachain.

## Ποια είναι η σύνδεση μεταξύ RWS και της Οικονομίας των Ρομπότ;

Όπως αναφέρθηκε προηγουμένως, ένας από τους στόχους πίσω από τα Robonomics Web Services είναι να εκπαιδεύσουμε τους χρήστες μας και να τους μετατρέψουμε σε συν-προγραμματιστές που κατανοούν την Οικονομία των Ρομπότ. Πιστεύουμε ότι δεν υπάρχει τίποτα περίπλοκο σε αυτό το έργο, αρκεί ο χρήστης να εξοικειωθεί με τη βασική λειτουργικότητα που συζητήθηκε παραπάνω.

### Ψηφιακά Δίδυμα

Το επόμενο βήμα στην εφαρμογή των εργαλείων που παρέχουν τα Robonomics Web Services είναι να μάθετε πώς να δημιουργείτε ένα Ψηφιακό Δίδυμο, γνωστό και ως Ψηφιακή Σκιά για την εφαρμογή. Λειτουργεί ως ψηφιακή εκδοχή του εξοπλισμού του πραγματικού κόσμου και αντιγράφει όλα τα τεχνικά του χαρακτηριστικά.

Τι σημαίνει αυτό, θα ρωτήσετε! Ας πάρουμε ως παράδειγμα μια περίπλοκη δομή αισθητήρων που μετρούν την ποιότητα του αέρα. Δεν μπορείτε να τους κάνετε να μετρούν κάθε 5 δευτερόλεπτα επειδή η εσωτερική τους βαθμονόμηση και η εξισορρόπηση των μετρημένων δεδομένων διαρκεί 2 λεπτά. Εδώ έρχεται το Ψηφιακό Δίδυμο, επιτρέποντάς μας να δημιουργήσουμε μια εφαρμογή που ανταποκρίνεται στις εντολές του χρήστη.

Σύμφωνα με την [έκθεση της Gartner για τις τάσεις της αγοράς IoT](https://www.gartner.com/en/innovation-strategy/trends/iot-adoption-trends), το 85% των εταιρειών που συμμετείχαν στη μελέτη έχουν ήδη εφαρμόσει ή σκοπεύουν να εφαρμόσουν Ψηφιακά Δίδυμα μέσα στον επόμενο χρόνο. Οι περισσότερες από αυτές τις εταιρείες διαθέτουν εσωτερικές ομάδες μηχανικών που εργάζονται σε ιδιόκτητες λύσεις. Παρ' όλα αυτά, όλες αναγνωρίζουν ότι πρέπει να βασίζονται σε ήδη υπάρχουσες λύσεις για να μπορούν να κλιμακώνονται γρήγορα και να διατηρούν τις υπηρεσίες τους προσαρμοσμένες στο συνεχώς μεταβαλλόμενο περιβάλλον της βιομηχανίας IoT.

Γι' αυτό το parachain μας στο Robonomics Network δεν προσφέρει μόνο τις προαναφερθείσες λειτουργίες που δεν είναι διαθέσιμες στο Ethereum IPFS, αλλά επιτρέπει επίσης στους χρήστες μας να δημιουργούν Ψηφιακά Δίδυμα των συσκευών τους.

Αναπτύσσουμε αυτή τη λειτουργία ώστε οι χρήστες να μπορούν να δημιουργούν τα δικά τους Ψηφιακά Δίδυμα, να τα ερμηνεύουν σε web services/IoT εφαρμογές και το σημαντικότερο να τα αποθηκεύουν ως μοντέλο που θα γράφει το datalog. Το [Wiki μας](http://wiki.robonomics.network/) ενημερώνεται συνεχώς με όλη την απαραίτητη γνώση για την υλοποίηση των παραπάνω λύσεων και σύντομα θα είναι πλήρες με βασικές πληροφορίες που εξηγούν την κατασκευή ενός Ψηφιακού Διδύμου από το Α έως το Ω.

Αναπτύσσουμε αυτό το χαρακτηριστικό για να επιτρέψουμε στους χρήστες να δημιουργούν τα δικά τους Ψηφιακά Δίδυμα, να τα ερμηνεύουν σε υπηρεσίες διαδικτύου/εφαρμογές IoT και, το πιο σημαντικό, να τα αποθηκεύουν ως μοντέλο που θα γράφει τα δεδομένα. Το [Wiki μας](http://wiki.robonomics.network/) ενημερώνεται συνεχώς με όλες τις απαιτούμενες γνώσεις για την εκτέλεση των προαναφερθέντων λύσεων και σύντομα θα είναι πλήρες με βασικές πληροφορίες που εξηγούν τη δημιουργία ενός Ψηφιακού Διδύμου από το Α ως το Ω.

Δίδυμα

Μόλις ολοκληρωθεί το μοντέλο επεξεργασίας δεδομένων και το Ψηφιακό Δίδυμο σας είναι πλήρως δομημένο, πρέπει να αποφασίσετε ποια θα είναι η σχέση του με την υπηρεσία web ή μια εφαρμογή IoT.

Η ανάλυση δεδομένων είναι συχνά η συνηθισμένη επιλογή για τέτοιου είδους ψηφιακές υπηρεσίες, όπου ένας αισθητήρας σας στέλνει τα δεδομένα που μετρά (ενημερώνει το ψηφιακό του αντίγραφο). Αυτά τα δεδομένα πιθανότατα θα υποβληθούν σε μεταγενέστερη επεξεργασία, καθώς είναι πολύ απίθανο να σας ενδιαφέρει η τιμή της πίεσης σε Πασκάλ μέσα στο σύστημα θέρμανσης του εξοχικού σας. Αντίθετα, χρειάζεστε να γνωρίζετε αν η θερμοκρασία στο σπίτι είναι σωστή και αν το σύστημα θέρμανσης λειτουργεί ακόμα. Ουσιαστικά, πρέπει να αποφασίσετε πώς θα εφαρμόσετε το ψηφιακό δίδυμο των δεδομένων που αποθηκεύσατε σε αυτό και εδώ είναι που μπορούν να βοηθήσουν πρόσθετες υπηρεσίες.

Τώρα, μεταβαίνουμε σε ένα σύνθετο σενάριο Εξισορρόπησης Έξυπνων Κτιρίων, όπου ένα αντίγραφο των δεδομένων κατανάλωσης ενέργειας λειτουργεί ως ψηφιακό δίδυμο. Ας υποθέσουμε ότι θέλετε να αποθηκεύσετε δεδομένα κατανάλωσης ενέργειας για ένα συγκεκριμένο διαμέρισμα και να χρησιμοποιήσετε τη λειτουργία Datalog με αποθήκευση IPFS για αυτό. Αν θέλετε να μάθετε το συνολικό ποσό της καταναλωθείσας ενέργειας, θα το αποθηκεύατε στην αλυσίδα. Αντίθετα, φανταστείτε ότι λειτουργούμε ένα ψηφιακό δίδυμο του έξυπνου κτιρίου. Η Υπηρεσία Εξισορρόπησης θα ζητά συχνά από αυτό το ψηφιακό δίδυμο να παρέχει διάφορα δεδομένα σχετικά με την κατανάλωση ενέργειας, στη συνέχεια θα τα αποθηκεύει και αφού πραγματοποιήσει υπολογισμούς εκτός αλυσίδας, θα συνδεθεί/επικοινωνήσει με το Robonomics για να παρέχει τις απαιτούμενες υπηρεσίες.

Συγκεντρώσαμε τέτοιες υπηρεσίες σε μια εξειδικευμένη [αγορά](https://robonomics.app/#/services) που λειτουργεί ως App Store για ρομπότ.

## Κατανόηση της Αγοράς
Για να υλοποιήσουμε ένα μοντέλο διασυνδεδεμένων ψηφιακών διδύμων και υπηρεσιών, πρέπει να δημιουργήσουμε ένα μέσο παράδοσης ή με άλλα λόγια μια εφαρμογή στο smartphone σας. Αυτή η αγορά, που βασίζεται σε ένα αποκεντρωμένο καθολικό με τη μορφή αποκεντρωμένων εφαρμογών, μπορεί να χρησιμοποιηθεί για την αντιγραφή και δημιουργία ατομικών αντιγράφων και ψηφιακών διδύμων για οποιονδήποτε απαιτούμενο σκοπό μέσω προκαθορισμένων προτύπων. Το σημαντικότερο είναι ότι αυτό γίνεται χωρίς μεσάζοντες και εξωτερική αποθήκευση, καθιστώντας την αγορά υπηρεσιών IoT μας ελεύθερη από λογοκρισία και κεντρικούς συνδέσμους.

Τέλος, ας μιλήσουμε για το περιβάλλον χρήστη. Στοχεύουμε πάντα να προσφέρουμε στους χρήστες μας τις πιο ενημερωμένες διαδικτυακές τεχνολογίες και επομένως παρέχουμε πρόσβαση σε αποκεντρωμένες εφαρμογές. Το σημαντικότερο χαρακτηριστικό που αποκτάτε κατά την ανάπτυξη τηςη Dapp σας, είναι η απουσία ευπαθειών από τη δική σας πλευρά. Ενώ η εφαρμογή σας βρίσκεται στο IPFS, είναι συνδεδεμένη με το μητρώο ENS ή το μητρώο του οικοσυστήματος Polkadot και έτσι ο χρήστης δεν αντιμετωπίζει την ευπάθεια να ασχοληθεί με μια υποκατεστημένη εφαρμογή ή ένα αίτημα όταν ακολουθεί τον υπερσύνδεσμο. Είναι απλά πολύ βολικό από την άποψη της ανάπτυξης εφαρμογών για τον τελικό χρήστη και της συντήρησής τους.

Ένα ακόμη πλεονέκτημα των αποκεντρωμένων εφαρμογών είναι η επικοινωνία peer-to-peer. Με άλλα λόγια, η εφαρμογή σας μπορεί να επικοινωνεί με το ψηφιακό της δίδυμο, την υπηρεσία IoT και την εφαρμογή του τελικού χρήστη χωρίς κανένα σημείο πτώσης σήματος, εξαλείφοντας την ανάγκη για τρίτους και ενδιάμεσους πόρους όταν πρέπει να εκτελεστεί μια εργασία. Αυτός είναι ο σκοπός του [robonomics.js](https://github.com/airalab/robonomics-js), που χρησιμοποιείται στην ανάπτυξη του [Dapp Robonomics](https://robonomics.app/#/). Αυτή τη στιγμή βρισκόμαστε στη διαδικασία προσθήκης μιας νέας σειράς βελτιώσεων και σύντομα θα κυκλοφορήσουμε το Robonomics Stylebook (έναν οδηγό διεπαφής βασισμένο στις λειτουργίες Web3.js και polkadot.js), αφήνοντάς σας με μια εύκολη δουλειά για να δομήσετε σωστά την εφαρμογή σας. Ως αποτέλεσμα, ολοκληρώνουμε ολόκληρη την αλυσίδα της Οικονομίας των Ρομπότ.

## Περίληψη
* Στόχος μας είναι να κάνουμε τα [ρομπότ μέρος της οικονομίας](https://blog.aira.life/the-first-in-the-economy-of-robots-6f4bb115643b) αλλά πρώτα, πρέπει να διδάξουμε στον χρήστη πώς λειτουργούν όλα και να τα ρυθμίσει.
* Για να βοηθήσουμε τον σκοπό αυτό, δημιουργήσαμε το προϊόν [Robonomics Web Services](https://blog.aira.life/robonomics-web-services-and-rws-token-intro-d730ab50ad42) που επιτρέπει στους χρήστες να δημιουργούν εφαρμογές και υπηρεσίες μαζί μας (συνδρομή, εκκίνηση, datalog).
* Η συνδρομή στο RWS επιλύει όλα τα ζητήματα που σχετίζονται με συναλλαγές και προσθέτει λειτουργίες που περιλαμβάνουν: ενεργοποίηση/απενεργοποίηση συσκευής, τηλεμετρία (αποθήκευση ψηφιακού διδύμου).
* Μόλις δημιουργήσετε ένα ψηφιακό δίδυμο μέσω εσωτερικών προτύπων, μπορείτε να αναπτύξετε μια υπηρεσία IoT και να την προσθέσετε στο [marketplace](https://robonomics.app/#/services) μας. Οι προδομημένες διεπαφές στο Dapp χρησιμοποιούνται ως μηχανισμός παράδοσης που φέρνει την υπηρεσία σας στον τελικό χρήστη.
* [Dapp](https://robonomics.app/) λειτουργεί ως ένας αδιάβλητος τρόπος για να καθιερώσετε μια επικοινωνία peer-to-peer (άμεση) με ένα ψηφιακό δίδυμο, ακόμα και όταν η υπηρεσία σταματήσει να λειτουργεί. Ανεξάρτητα από τις εξωτερικές συνθήκες, η αγορά δεν θα αποτύχει ποτέ να λειτουργήσει.
* Η προαναφερθείσα αλυσίδα αποτελεί μια διαδρομή προς την Οικονομία των Ρομπότ που περιγράφεται σε αυτό το άρθρο.
