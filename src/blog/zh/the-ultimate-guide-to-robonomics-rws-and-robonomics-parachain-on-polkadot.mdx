考虑到关于 Robonomics 核心运作的无休止讨论，让我们来探究其底层结构，并梳理出网络的主要支柱。本文将深入分析 Robonomics 平台、其在 [Polkadot 生态系统中的平行链](https://blog.aira.life/robonomics-parachain-on-kusama-40853780c709) 以及 [Robonomics Web Services](https://blog.aira.life/robonomics-web-services-and-rws-token-intro-d730ab50ad42)（RWS）之间的联系。

Robonomics 网络由多个[代码仓库](https://github.com/airalab)组成，这些代码可以单独用于实现一个功能完善的物联网应用。例如，如果你打算将智能手机连接到某个物联网设备，仅仅开发一个应用并连接一堆传感器是远远不够的。相反，你需要执行一系列[操作链](https://blog.aira.life/simple-iot-scheme-990224363803)，最终实现物联网设备的互联互通，使设备的信号能够顺利传递到你手中。

## Robonomics 网络发展的时间线

那么，创建 [Robonomics 平行链](https://parachain.robonomics.network/#/explorer) 和 [Robonomics 网络](https://robonomics.network/land/say-hello-to-robonomics/) 的想法是如何产生的呢？最初，在 2015–2017 年间，我们开始开发一个相当复杂的我们称之为[机器人经济](https://blog.aira.life/the-first-in-the-economy-of-robots-6f4bb115643b)的解决方案。我们坚信，这正是智慧城市、工业4.0以及物联网应用发展的必然趋势。

我们的首要目标是为网络物理系统开发软件，使其能够在技术和经济交易框架内管理自主和独立的设备。因此，我们决定生成一种兼具技术和经济属性的交易，并以此作为开发更复杂流程的基础。

我最喜欢的机器人经济场景之一被称为[智能建筑碳抵消](https://new.dapp.ipci.io/#/offsetting)。这是一项服务，允许一座自主建筑收集并汇总其能源消耗信息，然后将其传递给Robonomics Network中的另一个代理，甚至可以请求碳足迹抵消服务。随后，代理会验证建筑的位置，计算适用的二氧化碳排放系数，并将这些数据与合作项目[DAO IPCI](http://ipci.io/)同步，以确定需要从绿色金融市场购买多少碳信用额度来抵消该建筑的碳足迹。这些操作已在以太坊区块链上成功执行。

### 问题的解决方案

如今，我们与同事和合作伙伴在2017年构建的机器人经济，已经形成了一条乍看之下复杂的链条，成为物联网产业进化的最终阶段。我们面临的最大问题是，虽然积累了大量知识并实现了我们的想法，却没有机会将其传递给他人。每一个在Robonomics Network平台开发过程中创建的[机器人经济新场景](https://wiki.robonomics.network/docs/r-and-d-based-on-robonomics-network/)，都让最终产品对外行人来说变得更加难以理解。换句话说，必须具备多方面的能力，并花费数年时间，才能理解构成经济与机器协作概念的交叉学科背后的思维方式。

当你开始理解物联网应用开发背后的细节时，很快就会发现，技术专家需要花费数年时间才能达到产品完全可用的阶段。这也是我们决定将平台的潜在用户转变为共同开发者的原因。我们专注于教他们成为物联网应用领域的专业IT工程师，并且因此，我们创建了 Robonomics 平行链和 Robonomics Web Services。

## Robonomics Web Services 简介

很明显，机器人经济对公众来说很有吸引力。然而，主要问题在于，这些用户在短暂尝试后，仍然会转向 Amazon Web Services 来连接物联网设备，并开始使用它，而没有意识到 Robonomics Network 相较于知名 IT 巨头所具备的关键优势。这让我们得出结论：如果不能在当下提供真正的产品，仅仅承诺未来的某些成就，是远远不够的。

因此，我们决定必须为潜在用户提供一系列真实的服务。这些服务类似于其他云平台所提供的，但更为出色。最终，我们创建了 Robonomics Web Services，这是一款基于我们自有协议和平台、并采用开源机器人框架（ROS）构建的产品。随后，我们投入了两年时间，专注于关键功能的开发，接下来我将对这些功能进行详细介绍。

### RWS 的关键功能：

**1. 订阅**

区块链服务的标准模式总是包含被称为交易费用或 Gas 费的交易成本。这意味着，当你发起一笔交易时，其优先级取决于你愿意为其支付的费用。你支付的费用越高，交易处理的速度就越快。这样的模式并不适用于依赖于带宽保障的系统。你必须绝对确定你的设备能够在设定的时间、以可预测的成本完成交易。而在当下，这种保障并不常见，因为交易成本完全取决于汇率波动，有时甚至高达 50 美元。

为了解决这个问题，我们创建了订阅服务，这是一种在 Polkadot 上 Robonomics 平行链的替代成本模型。它保证预付数量的交易能够被执行，并且相较于通过费用模式支付的交易，享有更高的优先级。想要了解月度订阅与 XRT 代币之间的区别，可以参考我的《[异构通证经济学](https://blog.aira.life/rws-overview-part-2-heterogeneous-tokenomics-afc209cc855)》一文，而我计划中的另一篇文章将在十二月底发布的版本将概述这两种模型的最终架构及其互联方式。

我们的订阅模式不仅解决了保障过渡的问题，而且每台设备仅需10美元，使其成为终端用户极具盈利性的选择。此外，它为每月数千笔交易提供了稳定的带宽，这也是一个相当出色的成就。

**2. 启动**

从计算能力和区块链的角度来看，启动是一个非常简单且经济的功能。它允许你通过发送交易请求来开启或关闭你的设备，并将其二进制设置从1切换到0，反之亦然。由于该功能并不复杂，我们接下来进入下一个关键支柱。

**3. 数据日志（Datalog）**

此功能与遥测相关，因此需要我进一步说明。它有两种模式：你可以保存IPFS哈希，或将数据存储在链上。

那么如何使用这些模式？当你的数据量少于四个变量时，适用链上数据日志功能。将如此小的数据块保存在IPFS（链下）上并无意义，因为从字节数来看，这种链上日志与IPFS哈希相比并不会大多少。我们建议将所有原始数据流作为单一函数记录，或选择IPFS哈希。然后，Robonomics作为开发者平台会处理所记录的数据，无论是原始数据还是IPFS哈希，并决定将其写入何处。这个数据日志处理过程对终端用户是隐藏的，用户甚至不会察觉两种不同存储方式的区别。

“责任”是我们的最终功能，但在我们了解大规模机器人经济项目的目标之前，不会向公众发布。我们正在开发的另一个里程碑产品，旨在实现[机器人即服务](https://blog.aira.life/what-is-robonomics-network-robot-as-a-service-and-future-parachain-of-polkadot-or-something-more-36deece9f3cb)框架，或者我们称之为以太坊上的“合同义务机器人”。[该基础设施](https://blog.aira.life/robots-under-robonomics-control-overview-39337ac9fbbb)已在我们的知识中心中详细描述，意味着终端用户使用我们的服务代币无需依赖 XRT 或其他代币来支付平行链的使用费用。

## RWS 与机器人经济之间有什么联系？

如前所述，Robonomics Web Services 背后的目标之一是教育我们的用户，并将他们培养成了解机器人经济的共同开发者。我们相信，只要用户熟悉上文讨论的基本功能，这项任务并不复杂。

### 数字孪生

应用 Robonomics Web Services 提供的工具的下一步，是学习如何创建数字孪生，也称为应用的数字影子。它作为现实世界设备的数字版本，复制了所有技术特性。

你可能会问，这意味着什么？让我们以测量空气质量的复杂传感器结构为例。你不能让它们每 5 秒测量一次，因为它们的内部校准和数据平衡需要 2 分钟。这时，数字孪生就派上用场了，它让我们能够开发对用户指令有响应的应用。

根据 [Gartner 关于物联网市场趋势的报告](https://www.gartner.com/en/innovation-strategy/trends/iot-adoption-trends)，参与调研的公司中有 85% 已经实施或计划在未来一年内实施数字孪生。这些公司大多拥有自主研发团队，致力于专有解决方案。尽管如此，他们都承认必须依赖现有的解决方案，才能实现快速扩展，并保持服务对不断变化的物联网行业环境的适应性。

这就是为什么我们的 Robonomics Network 平行链不仅提供了以太坊 IPFS 上没有的上述功能，还允许用户为他们的设备创建数字孪生。

我们正在开发这一功能，让用户能够构建自己的数字孪生，在 Web 服务/物联网应用中解析它们，并且最重要的是，将其保存为可写入数据日志的模型。我们的 [Wiki](http://wiki.robonomics.network/) 正在不断更新，涵盖实现上述解决方案所需的全部知识，并很快将补充完整，提供从头到尾解释数字孪生构建的基础信息。

# 数字孪生

一旦数据处理模型确定并且您的数字孪生体完全构建好，您就需要决定它与 Web 服务或物联网应用之间的关系。

分析通常是此类数字服务的常见选择，在这种情况下，传感器会将其测量的数据发送给您（更新其数字副本）。这些数据很可能需要经过后处理，因为您很可能并不关心乡村别墅供暖系统内部的帕斯卡压力值。相反，您需要知道房屋内的温度是否合适，以及供暖系统是否仍在运行。基本上，您需要决定如何利用您保存的数据的数字孪生体，而这正是额外服务可以提供帮助的地方。

现在，我们转向一个更复杂的智能建筑碳抵消场景，其中能耗数据的副本充当数字孪生体。假设您想为某个公寓保存能耗数据，并为其使用带有 IPFS 存储的 Datalog 功能。如果您想了解总共消耗了多少能量，您会将其保存到链上。相反，假设我们运行的是智能建筑的数字孪生体。碳抵消服务会频繁地向这个数字孪生体请求各种能耗数据，然后将其存储，并在链下进行计算后，连接/与 Robonomics 进行通信，以提供所需的服务。

我们将此类服务汇集在一个专门的[市场](https://robonomics.app/#/services)中，它就像是机器人的应用商店。

![Robonomics 网络中的去中心化应用，dapp](/blog/images/the-ultimate-guide-to-robonomics-rws-and-robonomics-parachain-on-polkadot/robonomics-dapp.jpg)

## 理解市场

为了实现互联数字孪生体和服务的模型，我们必须为其创建一个交付媒介，换句话说，就是您手机上的一个应用。这个市场基于去中心化账本构建，以去中心化应用的形式存在，可以通过预定义模板为任何所需目的复制和创建个人副本及数字孪生体。最重要的是，这一切都无需中介和外部存储，使我们的物联网服务市场免受审查和中心化链接的影响。

最后，让我们谈谈用户界面。我们始终致力于为用户提供最新的在线技术，因此允许访问去中心化应用。在开发过程中，您将获得的最大特色是你的 Dapp 的另一个优势在于你这边没有漏洞。虽然你的应用部署在 IPFS 上，但它连接到 ENS 注册表或 Polkadot 生态系统注册表，因此用户在点击超链接时不会面临被替换应用或请求的风险。从终端用户应用开发和维护的角度来看，这非常方便。

去中心化应用的另一个优势是点对点通信。换句话说，你的应用可以与其数字孪生体、物联网服务以及终端用户应用进行通信，不存在信号衰减的单点问题，在需要执行任务时无需第三方和中介资源。这正是 [robonomics.js](https://github.com/airalab/robonomics-js) 的用途，它被用于开发 [Dapp Robonomics](https://robonomics.app/#/)。我们目前正在进行新一轮的改进，并很快会发布 Robonomics Stylebook（基于 Web3.js 和 polkadot.js 函数的界面指南），让你能够轻松地正确构建你的应用。最终，我们完成了完整的机器人经济链条。

## 总结
* 我们的目标是让[机器人成为经济的一部分](https://blog.aira.life/the-first-in-the-economy-of-robots-6f4bb115643b)，但首先，我们需要教会用户一切的工作原理并进行设置。
* 为了实现这一目标，我们创建了 [Robonomics Web Services](https://blog.aira.life/robonomics-web-services-and-rws-token-intro-d730ab50ad42) 产品，允许用户与我们一起构建应用和服务（订阅、启动、数据日志）。
* 订阅 RWS 解决了所有与交易相关的问题，并增加了包括：设备开/关、遥测（数字孪生体存储）等功能。
* 一旦你通过内置模板创建了数字孪生体，就可以开发物联网服务并将其添加到我们的[市场](https://robonomics.app/#/services)。Dapp 中预先结构化的界面作为交付机制，将你的服务带给终端用户。
* [Dapp](https://robonomics.app/) 作为一种坚不可摧的方式，即使服务停止运行，也能与数字孪生体建立点对点（直接）通信。无论外部环境如何变化，市场都将始终保持运作。
* 上述链路是本文所述机器人经济的通道。
